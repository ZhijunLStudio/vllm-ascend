# speculative_token_tree 实现方案与验证报告

## 一、参数说明

`speculative_token_tree` 是 `--speculative-config` 的子参数，用于配置 EAGLE 推测解码的 tree 结构。

### 与原生 vLLM 的关系

原生 vLLM 已完整支持 `speculative_token_tree`：
- `vllm/v1/attention/backends/tree_attn.py` — GPU tree attention backend
- `vllm/v1/spec_decode/eagle.py` — `propose_tree` 方法

NPU 需要实现对应的 backend 和适配。

---

## 二、NPU 实现方案

### 2.1 核心差异

| 层面 | GPU (原生 vLLM) | NPU (本实现) |
|------|----------------|-------------|
| Attention kernel | FlashAttention | `npu_fused_infer_attention_score` |
| Mask 格式 | float32 (0=attend, -inf=block) | int8 (0=attend, 1=block) |
| Mask 尺寸 | 原始 tree_len × tree_len | pad 到 2048×2048 |
| Mask 缓存 | 无 | 预计算所有 slice mask |
| ACL Graph | CUDA Graph | `ACLGraphWrapper` + `cudagraph_dispatcher` |

### 2.2 文件结构

```
vllm_ascend/
├── attention/
│   ├── backends/
│   │   ├── __init__.py          # backends 包初始化
│   │   └── tree_attn.py         # AscendTreeAttentionBackend
│   ├── attention_v1.py          # 注册 TREE_ATTN backend
│   └── attention_mask.py        # mask 转换工具
├── spec_decode/
│   └── eagle_proposer.py        # propose_tree + _propose_tree
└── ascend_forward_context.py    # slot_mapping 参数
```

### 2.3 AscendTreeAttentionBackend

```python
class AscendTreeAttentionBackend(AttentionBackend):
    get_name()         → "TREE_ATTN"
    get_impl_cls()     → AscendTreeAttentionImpl
    get_builder_cls()  → AscendTreeAttentionMetadataBuilder
```

### 2.4 AscendTreeAttentionMetadataBuilder

- 解析 `speculative_token_tree` 配置
- 构建 GPU float32 tree mask → 转换为 NPU int8 mask
- 预计算所有 slice mask（`_precompute_slice_masks`）
- `build_for_drafting`: 使用缓存的 mask，避免运行时重复计算

### 2.5 propose_tree / _propose_tree

- `_propose_tree`: 第一次 draft model forward + compute logits + 调用 `propose_tree`
- `propose_tree`: tree 层级循环，每层调用 draft model forward
- 每层支持 ACL Graph dispatch（`cudagraph_dispatcher.dispatch`）

## 四、Bug 修复记录

| 编号 | 严重度 | 问题 | 修复 |
|------|--------|------|------|
| 1 | P0 | `propose_tree` 从未被调用 | `_propose` 添加 `AscendTreeAttentionMetadata` 类型检查 |
| 2 | P0 | Backend 名称不匹配 | `"ASCEND_TREE_ATTN"` → `"TREE_ATTN"` |
| 3 | P0 | Root token 行为语义反了 | `tree_attn_mask[0, :] = 0` → `[:, 0] = 0` |
| 4 | P1 | `slot_mapping` 未传递 | `set_ascend_forward_context` 添加参数 |
| 5 | P1 | ACL Graph per-level 缺失 | `propose_tree` 添加 dispatch |
